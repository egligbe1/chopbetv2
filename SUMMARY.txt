📊 ARCHITECTURE SUMMARY
═══════════════════════════════════════════════════════════════════════════════

PREDICTION PIPELINE:
  
  [7:00 AM UTC] generate_predictions()
    ├─ Search for football fixtures
    ├─ Extract using Gemini
    ├─ Per-match analysis (form, injuries)
    └─ Save to DB with sport="football"
    
  [7:05 AM UTC] generate_nba_predictions()
    ├─ Search for NBA fixtures  
    ├─ Extract using Gemini
    ├─ Per-match analysis (lineups, injuries)
    └─ Save to DB with sport="basketball"
    
  [11:00 PM UTC] check_results()
    ├─ Query pending predictions (both sports)
    ├─ Search for final scores
    ├─ Evaluate outcomes
    └─ Update accuracy stats (per sport)


FRONTEND FLOW:

  Home Page (/ route)
    ├─ Sport toggle: Football ↔ Basketball
    ├─ Fetches /predictions/today?sport={football|basketball}
    └─ Displays predictions grouped by league
    
  Stats Page (/stats route)
    ├─ Sport toggle: Football ↔ Basketball
    ├─ Charts showing accuracy trends
    └─ Per-league & per-market breakdowns
    
  Admin Page (/admin route)
    ├─ Requires ADMIN_API_KEY authentication
    ├─ Three trigger cards:
    │   ├─ Football Predictions (primary color)
    │   ├─ NBA Predictions (orange #F58426)
    │   └─ Check Results (secondary color)
    └─ Background task execution


DATABASE SCHEMA:

  Predictions Table
    ├─ id (PK), date, home_team, away_team
    ├─ league, country, sport ← "football" or "basketball"
    ├─ kickoff_time, market, prediction
    ├─ confidence (0-100), odds, risk_rating
    ├─ reasoning, source_link
    ├─ status ("pending" | "won" | "lost" | "void")
    └─ created_at, FK to Results
    
  Results Table
    ├─ id (PK), prediction_id (FK)
    ├─ ht_score_home, ht_score_away
    ├─ ft_score_home, ft_score_away
    └─ result_checked_at
    
  AccuracyStats Table
    ├─ date, sport
    ├─ total_predictions, correct, incorrect
    ├─ accuracy_pct
    ├─ by_league (JSON breakdown)
    └─ by_market (JSON breakdown)
